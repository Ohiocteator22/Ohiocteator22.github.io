Plugins / Plugin API Tutorial
Guide

Writing a CornUi plugin

CornUi plugins are just Lua modules loaded at runtime and handed a shared context object. Broken plugins can't take down the hub, since every plugin call runs inside a pcall.

Loading a plugin

Plugins are hosted as raw source files (usually on GitHub) and loaded the same way CornUi itself is loaded:

local Corn = loadstring(game:HttpGet('YOUR_RAW_CORNUI_URL'))() local Window = Corn:CreateWindow({ Name = 'Corn Hub' }) -- load a single plugin Corn:LoadPlugin('YOUR_RAW_PLUGIN_URL') -- or load several at once Corn:LoadPlugins({ 'YOUR_RAW_PLUGIN_URL_1', 'YOUR_RAW_PLUGIN_URL_2', })

Each plugin call is wrapped in a pcall internally, so if a plugin errors on load, it just fails quietly instead of taking the rest of the hub down with it.

The plugin file itself

A plugin is a Lua chunk that returns a function. That function receives a shared ctx (context) table when the plugin loads, giving it access to the window and library internals it needs to add its own tabs, sections, and elements.

-- MyPlugin.lua return function(ctx) local Window = ctx.Window local Library = ctx.Library local Tab = Window:CreateTab({ Name = 'My Plugin' }) local Section = Tab:CreateSection({ Name = 'General' }) Section:CreateToggle({ Name = 'Enabled', Flag = 'MyPlugin_Enabled', Callback = function(value) -- your logic here end }) end
Placeholder: the exact fields on ctx (Window, Library, Flags, etc.) should match whatever CornUi.lua actually exposes — update this block if the real signature differs.

Flags

Any element created with a Flag automatically syncs into Library.Flags, and can be read or written from anywhere with GetFlag / SetFlag, or watched through the FlagChanged event — the same system the core library elements use.

Good practices

  • Keep it self-contained. Don't assume other plugins are loaded unless you check for them first.
  • Guard your callbacks. The outer pcall protects the load step, not necessarily every callback fired later — wrap risky logic in your own pcall too.
  • Namespace your flags. Prefix flag names with your plugin's name to avoid collisions with the core library or other plugins.
